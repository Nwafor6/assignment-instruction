How and What to do in Lesson Assignments (Parts 1 and 2 of Lessons 1, 2, 3)

Understand the contents of the lecture videos

  • Understand the purpose of every singleline of the code
  • If it does not make perfect sense, pause, and ask questions (via D2L Q&A forum).

Produce the same code after understanding them. If possible, without directly looking into the video.

  • If you mindlessly copy the code as shown in the video, you will develop excellent “typing skills.” However, if you reproduce the code by yourself after understanding them without directly looking into the videos, you will develop good “programming skills.”
  • As you know, I am unable to tell how your submitted code was produced. You may get a good grade for your good typing skills. You can choose what you want to be in the future; a typist or a programmer.

When an error occurs:

  • If your dream is to be a super good typist rather than a programmer, Looking hard into videos to compare it with your code and find the difference.
  • If you want to take this opportunity to improve programming skills,
    1. Mostly, it is due to a typo that you have overlooked.
    2. In case of compile errors, understand the meaning of the error message, locate the cause of the errors, and fix them.
    3. In case of logic errors (incorrect results), use the debugger to trace the program execution and locate the cause of logic errors.
    4. Troubleshooting bugs using error messages is a very important programming skill.

You may add more code of your own to further practice programming. In this case:

  • First, complete the identical code shown in the lecture video.
  • Then, you may add your own code to deepen your understanding of the topic.
  • If you have added your own code, put them in “comments” before you submit the assignment. That is, when your program runs, it should produce the same output as that of video lecture code.

If you rush to complete the lesson assignments

  • If you rush to complete the lesson assignments without spending enough time to contemplate each line of code, you do not learn much out of lesson assignments.
  • After completing the lesson assignments, each line of code (syntax, meaning, and purpose) must make sense to you.

Grading policy

  • For each assignment, both the program and the video demo(s) must be submitted to receive credits. No partial credit if not all is submitted
  • The deployed app should also work. No partial credit may be given if the app works only on the local emulator.
  • If the program produces compile errors or runtime errors, zero credits will be given without any partial credits.
  • Any difference (except the comments) between the submitted code and lecture node is subject to penalty.
  • Late submission is accepted within the next 12 hours after the due date/time with an hourly late penalty (3 points per hour)
  • Late submission is determined by whichever is submitted late (program or video demos).
